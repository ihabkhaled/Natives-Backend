## What was intentionally NOT created (recorded, not omitted)

Per `06-technical-refinement.md`: no new top-level `knowledge/`, `security/`, `privacy/`, `operations/`, `product/`, `domain/`, `contracts/`, `quality/`, `incidents/`, or `structure/` directory (each is already owned or genuinely not applicable); no AST/embeddings/watcher/static-pack machinery; no call/contract/event/deployment graph (no event bus, no IaC, no on-disk OpenAPI exist in this repo). The privacy/operations gaps were filled as single `memory/*-decisions.md` files that explicitly mark their not-applicable sections, and the contracts concern as a routing-only `context/` map — extending existing patterns rather than inventing parallel trees.

## Remaining documentation gaps

None blocking. Future extensions documented in `06-technical-refinement.md`: per-module `.ai/summaries/*.md` cards (once `BOOTSTRAP.md` exceeds budget), an OpenAPI-JSON-backed API contract manifest, and an event-graph (the moment `src/core/events/` exists — `modules.json`'s `hasEvents` field already signals it).
