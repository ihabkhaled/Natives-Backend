# 26 — Hypercare report

Hypercare has not started because the feature is not released. When released,
monitor:

- pending/retrying/dead-letter outbox counts and delivery failures;
- reminder preview counts versus dispatched counts;
- calendar generic-unavailable rates without recording bearer tokens;
- token issue/revoke volume and active-token limit errors;
- audience-cap events requiring explicit segmentation.

Use the practice notification/calendar runbook for replay and credential
incidents.
