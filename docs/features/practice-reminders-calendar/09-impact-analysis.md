# Impact analysis

- Backend/API: new calendar feed, quiet-hours, and admin preview/test endpoints.
- Frontend/mobile: generated OpenAPI consumers will need optional UI later; existing contracts are
  additive.
- Database: two additive tables and indexes; no backfill.
- Auth/RBAC: protected routes reuse `practice.read`, `notification.preferences.self`, and
  `jobs.manage`; public feed is authorized solely by an opaque credential plus active membership.
- Privacy: feed and logs omit notes, participant lists, contact data, raw tokens, and provider data.
- Async/ops: existing outbox retry/dead-letter metrics and replay are reused.
- Monitoring/support: document token revoke, preview/test, outbox metrics, and replay.
- CI/contracts: OpenAPI changes must be regenerated by the owning prompt-900 pipeline after this
  slice settles.
- Compliance/legal: no new sensitive-data category or external processor.
- Training: support/admin guidance is required; end-user UI is outside backend scope.

Backward compatibility is additive. Migration rollback removes only newly created data.
