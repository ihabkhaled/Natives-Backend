## What this is

This file is the permanent operating brain for AI-assisted delivery. It is not a feature document. It is the standing company policy that defines how software work must be classified, analyzed, planned, implemented, validated, documented, released, monitored, and improved.

## Authority (read in this order when guidance conflicts)

1. `claude.md` — canonical governance policy.
2. `rules/00-non-negotiable-rules.md` + `context/architecture-map.md` — the engineering canon.
3. The layer rule for what you're touching (`rules/README.md`).
4. `AGENTS.md` / `.cursor/rules/*.mdc` / `.cursorrules` / `codex.md` / `cursor.md` / the 5 family entrypoints — compatibility mirrors, never richer than `claude.md`.

## Fast task protocol

1. Run `npm run knowledge:context -- --task="<what you're about to do>"`.
2. Read `.ai/local/current-context.md` — the warm-up set plus ranked rules/skills/source for this task.
3. Read the exact source and tests it names before planning.
4. Run the Simple Code Ladder (`rules/20-simple-readable-code.md`) before writing new code.

## Quality gates (all green before done)

```bash
npm run lint
npm run typecheck
npm run test:coverage
npm run build
```

Never bypass a hook with `--no-verify`. A red gate is fixed at the root cause, never suppressed.
